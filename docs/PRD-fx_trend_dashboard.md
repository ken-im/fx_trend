# FX Trend Dashboard 요구사항정의서 (초안)

## 1. 개요

### 1.1 프로젝트 명
- **FX Trend Dashboard**
- 한글명: 환율 트렌드 대시보드

### 1.2 추진 배경 및 목적
본 프로젝트는 Python 기반의 **FinanceDataReader(FDR)** 라이브러리를 활용하여 주요 통화(USD, JPY, EUR 등)의 환율 데이터를 수집하고,
환율 추세 및 이동평균(Moving Average) 지표를 시각화함으로써 환율 변동 흐름을 직관적으로 파악할 수 있는 대시보드를 구축하는 것을 목적으로 한다.

개인 GitHub 공개 및 향후 서비스 확장을 고려한 구조로 설계한다.

### 1.3 기대 효과
- 환율 시계열 데이터에 대한 직관적 이해
- 이동평균 기반 중·장기 추세 파악
- 금융 데이터 시각화 포트폴리오 확보

## 3. 데이터 수집 요구사항

### 3.1 대상 통화
- USD/KRW (미국 달러)
- JPY/KRW (일본 엔화)
- EUR/KRW (유로화)
- 향후 통화 확장 가능 구조로 설계

### 3.2 데이터 소스
- FinanceDataReader 제공 환율 데이터

### 3.3 수집 조건
- 기간: 사용자 입력 또는 기본값(최근 5년) 제공
- 주기: 일 단위 (Daily)

### 3.4 데이터 처리
- 결측치 처리
- 날짜 기준 정렬
- 통화별 데이터 구조 표준화

---

## 4. 분석 기능 요구사항

### 4.1 환율 추세 분석
- 통화별 환율 시계열 그래프 제공
- 기간 내 최고/최저 환율 표시 (선택)

### 4.2 이동평균 분석
- 3개월 이동평균: 60일간의 환율 평균, 표시기호=MA3M
- 1년 이동평균: 250일간의 환율 평균, 표시기호=MA1Y
- 3년 이동평균: 750일간의 환율 평균, 표시기호=MA3Y

### 4.3 이동평균 계산 방식
- 단순 이동평균(Simple Moving Average, SMA)
- 계산 공식:
  - SMA(n) = n일간 환율 평균

---

## 5. 시각화 요구사항

### 5.1 기본 그래프
- X축: 날짜(Date), 년도 및 분기(보조선) 간격 표시(예시: 2025-2Q)
- Y축: 환율
- 환율 원본 데이터 + 이동평균선 중첩 표시
- 선택 기간 기준 최고점 일자, 최저점 일자 및 최근 일자의 원본 환율 값을 표시
- 현재 일자의 원본 환율 값은 수직선 추가하여 강조

### 5.2 그래프 옵션
- 통화 선택: 기본값=USD/KRW (미국 달러)
- 기간 선택: 기본값=5년
- 이동평균선 On/Off

### 5.3 UI/UX
- 반응형 레이아웃
- 통화 및 기간 선택 드롭다운 제공

## 8. GitHub Pages 배포 요구사항

### 8.1 배포 목적
- Python으로 분석·생성된 환율 트렌드 및 이동평균 결과를 **GitHub Pages를 통해 정적 웹 형태로 제공**한다.
- 개인 GitHub 포트폴리오 및 외부 공유를 주요 목적으로 한다.

### 8.2 배포 방식
- GitHub Pages의 정적 호스팅 특성을 고려하여 **서버 사이드 Python 실행은 포함하지 않는다.**
- Python은 데이터 수집·분석 및 HTML 생성 용도로만 사용한다.

### 8.3 기술 구성
- 데이터 수집/분석: Python + FinanceDataReader
- 시각화 결과물: HTML 파일
- 배포 채널: GitHub Pages
- 시각화 라이브러리: Plotly

### 8.4 HTML 생성 요구사항
- 환율 추세 및 이동평균 그래프는 **단일 HTML 파일(index.html)** 로 생성한다.
- HTML 파일은 외부 서버 의존 없이 단독 실행 가능해야 한다.
- 통화별 그래프는 '5.2 그래프 옵션'에 따라 보여준다.

### 8.5 콘텐츠 구성
- 페이지 상단: 프로젝트 명칭 (FX Trend Dashboard)
- 요약 영역: 최고 환율 / 최저 환율 / 현재 환율 정보 표시
- 메인 영역: 환율 추세 + 이동평균 시각화 그래프
- 하단 영역: 데이터 출처 및 생성 일자 표기

### 8.6 업데이트 전략 (선택)
- Python 스크립트를 통해 정기적으로 HTML 파일을 재생성할 수 있는 구조를 고려한다.
- GitHub Actions를 활용한 자동 업데이트 구조는 향후 확장 범위로 정의한다.
